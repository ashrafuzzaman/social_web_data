API

These are the REST api documentation of the data service for users. The project source is hosted in https://github.com/ashrafuzzaman/social_web_data. And the service is hosted in http://social-web-data.heroku.com
Fetch friends status
Path /statuses/friends_status.json
HTTP method GET
Params
email the email of the user
friends_email the email of the friend
shared_key the key that was shared while accepting as friend
Description Returns the status list of a friend. This is requested in the friends data server.
Fetch own status
Path /statuses.json
HTTP method GET
Params
email the email of the user
auth_token the auth_token is generated while login
Description Returns the status list of the user. This is requested in the user's own data server.
Create status
Path /statuses.json
HTTP method POST
Params
email the email of the user
auth_token the auth_token is generated while login
status the status text
Description This is used to create the status of user in their own data server
Fetch status information
Path /statuses/{:id}.json
HTTP method GET
Params
email the email of the user
auth_token the auth_token is generated while login
id the id of the status in database
Description This is used to fetch status details
Handle friends request accepted
Path /friends/handle_friend_req.json
HTTP method POST
Params
email the email of the user
friends_email the email of the friend
shared_key the key that was shared while accepting as friend
Description After accepting fried request this is used to notify after accepting and to update to shared secret key.
It stores the friend requests in the users data base

HTTP method POST
Params
email the email of the user
friends_email the email of the friend
shared_key the key that was shared while accepting as friend
Description This creates a friend request notification, which latern on the user have to accept.

The cofirmation to accept friend request
Path /friends/accept_friend_req.json
HTTP method POST
Params
email the email of the user
auth_token the auth_token is generated while login
friend_with the email of the friend
Description This service confirms a friend request and adds in the network.
Notify the friend as we accept his request
Path /friends/accepted_friend_req.json
HTTP method POST
Params
email the email of the user
friend_with the email of the friend
Description This service notifyes the friend that the request is accepted. This is posted in friends server.

Create new profile attribute
Path /profile_attributes.json
HTTP method POST
Params
email the email of the user
auth_token the auth_token is generated while login
profile_attribute[name] attribute name
profile_attribute[value] the value of the attribute
Description Create new profile attribute in the user's data server.
Delete profile attribute
Path /profile_attributes/{:id}.json
HTTP method DELETE
Params
email the email of the user
auth_token the auth_token is generated while login
id id of the attribute
Description Deletes profile attribute from all the profiles

Post comment
Path /comments/post_comment.json
HTTP method POST
Params
email the email of the user
auth_token the auth_token is generated while login
resource_type which resource type is the comment is for, i.e. status
resource_id the resource identifier in the server the resource is in
sequence_number last sequence number that the comment has seen. Used to sort the comments
comment the comment text
Description Post comment in own status that is stored in the user's own server.
List of the comments that are posted by the user

HTTP method GET
Params
email the email of the user
auth_token the auth_token is generated while login
friends_email the email of the friend
resource_type which resource type is the comment is for, i.e. status
resource_id the resource identifier in the server the resource is in
Description List of the comments that are posted by the user which are stored in the user's own server.
